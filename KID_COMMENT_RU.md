# Объяснение как для ребенка

Представь, что у нас есть большой пульт управления играми.

Раньше каждая кнопка жила отдельно, и мы передавали команды из комнаты в комнату через много рук.
Это долго и легко запутаться.

Теперь мы сделали один общий "умный шкаф" с настройками. Этот шкаф называется Zustand.

## Что мы сделали по шагам

1. Сделали общий шкаф (`useGameQuery` + `Store`).
Теперь фильтры (жанр, платформа, поиск, сортировка) лежат в одном месте.

2. Научили загрузку игр (`useGame`, `GameGrid`) брать данные прямо из шкафа.
Больше не нужно таскать фильтры через `App`.

3. Научили кнопки и селекторы (`Nav`, `SearchBar`, `GenreList`, `GenreSelector`, `PlatformSelector`, `SortSelector`) работать прямо со шкафом.
Каждый компонент сам берет нужные данные.

4. Упростили `App.tsx`.
`App` теперь только рисует экран и не хранит состояние фильтров.
Внутри `App` больше нет React-хуков для фильтров.

5. Удалили старый ненужный тип `GameQueryParams`.
Он больше не нужен, потому что типы уже живут рядом со стором.

6. Проверили, что проект собирается.
Сборка прошла успешно.

## Итого

- Меньше передачи пропсов между компонентами.
- Логика фильтров собрана в одном месте.
- Код стал проще читать и поддерживать.
- `App` стал "чистым" контейнером экрана.
